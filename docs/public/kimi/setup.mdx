# Kimi AI Setup

> **Give Kimi AI persistent memory across sessions.**

Kimi starts every conversation fresh. Claude-mem changes that by capturing observations and decisions, then injecting relevant context back into each new session via `KIMI.md`.

<Info>
**How it works:** Claude-mem captures all kimi conversations by watching `~/.kimi/sessions/` in real time. It also redirects API calls through a local proxy (port `11451`) so `KIMI.md` context is injected into every session automatically.
</Info>

## Three Integration Lanes

<CardGroup cols={3}>
  <Card title="Lane A — HTTP Proxy" icon="server">
    Local proxy on port `11451`. Patches `~/.kimi/config.toml` and VS Code settings so all API calls are intercepted — injects `KIMI.md` as system context.
  </Card>
  <Card title="Lane B — CLI Wrapper" icon="terminal">
    Transparent bash wrapper at `~/.kimi-mem/bin/kimi`. Drop-in replacement for the standalone `kimi` CLI.
  </Card>
  <Card title="Lane C — Session Watcher" icon="eye">
    Watches `~/.kimi/sessions/*/wire.jsonl` for new turns. Captures conversations from both CLI and the VSCode extension with zero configuration.
  </Card>
</CardGroup>

## Prerequisites

- [Bun](https://bun.sh) runtime (`curl -fsSL https://bun.sh/install | bash`)
- Kimi CLI or Kimi VSCode extension (`moonshot-ai.kimi-code`) installed

## Installation

### Option 1: With Claude Code (recommended)

If you have Claude Code installed, add this to `~/.claude/settings.json`:

```json
{
  "extraKnownMarketplaces": {
    "kalimeromk": {
      "source": {
        "source": "github",
        "repo": "KalimeroMK/claude-mem"
      }
    }
  },
  "enabledPlugins": {
    "claude-mem@kalimeromk": true
  }
}
```

Then open Claude Code — it will install the plugin automatically. After that:

```bash
npx claude-mem install kimi
```

### Option 2: Without Claude Code

If you don't have Claude Code, use `bunx` to install directly from the fork:

```bash
bunx github:KalimeroMK/claude-mem install kimi
```

<Note>
**Prerequisites:** [Bun](https://bun.sh) and [Node.js 18+](https://nodejs.org) must be installed.

```bash
# Install Bun
curl -fsSL https://bun.sh/install | bash
source ~/.zshrc

# Install Node.js via nvm (if not already installed)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
nvm install --lts
```

After installing both, run `bunx github:KalimeroMK/claude-mem install kimi`.
</Note>

<Warning>
`npx claude-mem install` downloads the **original thedotmack package** from npm — it does not include the Kimi integration. Use `bunx github:KalimeroMK/claude-mem` or the `install.sh` script below.
</Warning>

### Option 3: One-liner install script

```bash
curl -fsSL https://raw.githubusercontent.com/KalimeroMK/claude-mem/main/install.sh | bash -s -- kimi
```

This checks for prerequisites, installs Bun if missing, and runs the installer automatically.

## Starting the Worker

The Kimi proxy starts with the claude-mem worker on port `37777`. Start it manually:

```bash
claude-mem worker start
```

Verify it's running:

```bash
curl -s http://localhost:37777/api/health
```

### Auto-start on login (macOS)

To have the worker start automatically on every login:

```bash
curl -fsSL https://raw.githubusercontent.com/KalimeroMK/claude-mem/main/autostart.sh | bash
```

This creates a macOS LaunchAgent (`~/Library/LaunchAgents/com.claude-mem.worker.plist`) that starts the worker on login and after reboots.

To remove auto-start:

```bash
curl -fsSL https://raw.githubusercontent.com/KalimeroMK/claude-mem/main/autostart.sh | bash -s -- remove
```

## Using the Integration

After `claude-mem install kimi`, everything is automatic:

1. **Start the worker** — `claude-mem worker start`
2. **Use kimi normally** — via the VS Code extension or CLI. Conversations are captured by the session watcher.
3. **Context is injected** — `KIMI.md` is updated after each turn and injected into future sessions via the proxy.

### How Lane A works (proxy + config patch)

`install kimi` patches `~/.kimi/config.toml` to redirect the kimi API through the local proxy:

```toml
# Before install:
base_url = "https://api.kimi.com/coding/v1"

# After install:
# claude-mem-original-base_url = "https://api.kimi.com/coding/v1"
base_url = "http://localhost:11451/v1"
```

It also sets `KIMI_BASE_URL` in VS Code settings as a belt-and-suspenders measure.

### Custom API clients

If you're calling the Moonshot API directly from code, point at the proxy:

```javascript
import OpenAI from 'openai';

const client = new OpenAI({
  apiKey: process.env.MOONSHOT_API_KEY,
  baseURL: 'http://localhost:11451/v1',
});
```

### CLI wrapper (Lane B)

A wrapper is also installed at `~/.kimi-mem/bin/kimi`. Restart your shell or:

```bash
source ~/.zshrc
```

Now `kimi` automatically routes through the memory proxy.

## How KIMI.md Works

`KIMI.md` lives at `~/.kimi/KIMI.md` (or `$KIMI_MD_PATH_OVERRIDE`). The proxy:

1. Reads `KIMI.md` at the start of each request
2. Injects it as a `system` message (only if no system message exists)
3. After each response, refreshes `KIMI.md` with updated context from the worker

The relevant section is wrapped in `<claude-mem-context>` tags so it can be updated without overwriting your own notes.

```markdown
# My Kimi Notes
These are my personal instructions...

<claude-mem-context>
# Recent Activity
...context injected here by claude-mem...
</claude-mem-context>
```

## Uninstall

```bash
claude-mem uninstall kimi
```

This removes the CLI wrapper, cleans the `<claude-mem-context>` block from `KIMI.md`, and removes the VSCode watcher entry. Your own content in `KIMI.md` is preserved.

## Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `CLAUDE_MEM_KIMI_PROXY_ENABLED` | `true` | Set to `false` to disable the proxy |
| `KIMI_MD_PATH_OVERRIDE` | `~/.kimi/KIMI.md` | Override KIMI.md location |
| `KIMI_CWD` | `process.cwd()` | Project directory (used if `X-Kimi-Cwd` header not set) |

## Troubleshooting

<AccordionGroup>
  <Accordion title="Conversations not being captured">
    Check that the worker is running: `curl -s http://localhost:37777/api/health`. The session watcher starts automatically with the worker and watches `~/.kimi/sessions/` — no extra config needed.
  </Accordion>
  <Accordion title="KIMI.md not being injected into conversations">
    The proxy must be running AND `~/.kimi/config.toml` must have the patched `base_url`. Verify: `grep base_url ~/.kimi/config.toml` should show `http://localhost:11451/v1`. If not, run `claude-mem kimi install` again.
  </Accordion>
  <Accordion title="Proxy returns 502 Bad Gateway">
    The kimi binary uses OAuth, not API keys. Make sure you're logged in: run `kimi login` in a terminal. The proxy forwards the OAuth token from `~/.kimi/credentials/`.
  </Accordion>
  <Accordion title="Worker not starting">
    Make sure Bun is installed: `bun --version`. Then check worker logs: `claude-mem worker logs`
  </Accordion>
  <Accordion title="CLI wrapper not found after install">
    Add `~/.kimi-mem/bin` to your PATH and reload your shell: `source ~/.zshrc`
  </Accordion>
  <Accordion title="Restoring original kimi config (removing claude-mem)">
    Run `claude-mem kimi uninstall`. This restores `~/.kimi/config.toml`, removes `KIMI_BASE_URL` from VS Code settings, and preserves all your kimi data.
  </Accordion>
</AccordionGroup>
